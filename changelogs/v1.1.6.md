## What's Changed in v1.1.6

### Onboarding Questionnaire
- **Application Form**: New users must now complete a short questionnaire before their account enters the admin review queue. This replaces the blank "Account under review" wall that had no context for admins.
- **Store Description**: Applicants describe what their store is about (required).
- **Store Type**: Optional chip selection — Hardware, Clothing, Food, Electronics, General, or Other.
- **Shop Link**: Applicants can share an existing social media page, website, or any shop link (optional).
- **Referral Source**: How the user found OmniShop — Friend, GitHub, Social Media, Our Page / Ad, or Other (required).
- **Location**: Optional store city or region field.
- **Admin Review**: Admins can now click **View Application** on any pending user card in the Approvals page to read the applicant's full answers in a modal before deciding to approve or reject.
- **Flow**: Sign-up → (verify email) → Onboarding questionnaire → Pending review → Approved → Dashboard.

### Socials — Messenger & Gmail
- **Embedded Messenger**: Live Facebook Messenger webview under `Socials › Messenger`, with a persistent session that survives app restarts.
- **Embedded Gmail**: Live Gmail webview under `Socials › Gmail`, with a persistent session partition.

### Store & Settings Overhaul
- **Redesigned Settings**: Switched to a cleaner two-panel layout with dedicated sections for Account, Appearance, Display, Notifications, System, and Danger Zone.
- **Dedicated Store Management**: Moved shop configuration to a new top-level page under `Store › Store Management`.
- **Members & Invite Codes**: Store owners can now invite team members using unique codes and manage their roles (Owner/Member).
- **Display Settings**: Added multi-monitor support for fullscreen mode. You can now select which screen to use.

### Sidebar Updates
- **Store Switcher**: Replaced the static header with a dropdown menu to easily see the active store and user role.
- **Improved Navigation**: Moved `Storage` into the `Store` group for better organization.
- **Role-Based Views**: Members see a simplified sidebar (hiding Sales/Finance) and read-only views for critical store settings.

### Header Updates
- **Online Status**: Moved the connectivity indicator to the global header toolbar for better visibility.
- **Router Updates**: Updated breadcrumbs to reflect the new Store Management structure.

### Fixes & Improvements
- **Member & Data Loading**: Fixed critical bugs where shop owners could not see their member list and new members could not see shop data (like Categories) on their first login.
- **Store Setup Crash**: Fixed a crash on the "Set up your shop" page when continuing to the business information step.
- **Webview Session Isolation**: Sessions for Messenger and Gmail are now isolated per user account, fixing a privacy issue where a new user could see the previous user's session.
- **Firebase Security Rules**: Updated security rules to correctly handle permissions for shop owners versus members.
- **Build Stability**: Fixed a duplicate function declaration error that was breaking the build.
- Fixed webview containers (Messenger/Gmail) not filling the full panel height.
- Fixed various linting issues and unused variables.
- Improved authentication flow for new store setups (Join vs Create).
- Enhanced code stability and type safety across the board.
